QUESTÕES BÁSICAS SOBRE O GIT


1 - Para que serve o commit e qual a melhor forma de utilizá-lo?


O comando git commit é uma das funções principais do Git, git commit é usado para criar um instantâneo das alterações preparadas em uma linha do tempo de um histórico de projetos do Git. Os commits são as unidades estruturais de uma linha do tempo de projeto Git. Os commits podem ser considerados instantâneos ou marcos ao longo da linha do tempo de um projeto Git. Os commits são criados com o comando git commit para capturar o estado de um projeto naquele momento. O commit dos instantâneos do Git sempre é feito no repositório local.


2 - Qual a diferença entre commit e push?


O commit fará simplesmente o registro de suas alterações feitas em sua máquina local. Não marcará a mudança no repositório remoto, enquanto git Push atualiza referências remotas junto com objetos associados. Sendo assim, o primeiro é usado em conexão com seu repositório local, enquanto o último é usado para interagir com um repositório remoto.

3 - O que é tag?


São etiquetas que demarcam um ponto, um commit que representa alguma mudança significativa no seu código, ou seja, uma versão do seu projeto. Pode ser criada também para apontar para um commit específico e, portanto, não muda, mesmo se a branch seguir em frente. Em outras palavras, tags são referências imutáveis. O Git tem dois tipos de tags: annotated e non-annotated, existe uma pequena diferença entre elas. Ambas permitem que você faça referência a um commit específico em um repositório.

4 - O que é branch?


Um branch consiste em um desenvolvimento, é uma ramificação do estado do código que cria um novo caminho para a evolução do mesmo. Ele pode estar em paralelo com outros branches realizados. Sendo, possível incorporar novas funcionalidades para o código de um jeito organizado. É possível desenvolver novos recursos para a aplicação sem impedir o desenvolvimento no branch principal. Com o branches, é possível criar diferentes branches de desenvolvimento que podem convergir no mesmo repositório. Por exemplo, um branch estável, um branch teste e um branch instável.


5 - Como resolver conflitos no Git?


Um conflito no Git ocorre quando dois desenvolvedores alteram o mesmo trecho de código e a única maneira de resolver este conflito é através de uma intervenção manual, alterando o código em questão e submetendo um novo commit. Conflitos podem acontecer tanto ao mesclar branches , quanto ao mesclar commits dentro da mesma branch. Conflitos no Git são bastante comuns e acontecem sempre quando o mesmo arquivo foi modificado por duas versões diferentes e essas versões não podem ser automaticamente mescladas, necessita-se localizar todos os arquivos onde ocorreram conflitos para a resolução do conflito.
